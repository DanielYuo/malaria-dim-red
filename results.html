Results

Seurat

When the dimensionality reduction was done on the unintegrated data, there seemed to be a clear separation between the infected and uninfected samples. However, within the samples of the same treatment (infected and uninfected) clear overlaps were seen. This can be seen in the Figure 1 below. This suggested that there are clear distinctions between the infected and uninfected samples.

When the data integration and batch correction were applied to the dataset, the dimensionality reduced plots showed that the samples are integrated with each other. However, here it can be seen that the data may have been overcorrected, such that the biological differences might be missing due to the process. This can be seen on the Figure 2 below, where there are clear overlaps between each condition across the entire plots.

tSNE Optimisation

When optimising the dimensionality reduction, the main parameter that was optimised for tSNE was the perplexity. This parameter determines the number of nearest neighbours attracted to each point by controlling the width of the gaussian kernels for the computation of the similarities between them [10].

Based on this, when the perplexity was set to the value of one, almost all points in the plot have the same gap and evenly distributed to an almost circular shape, which can be seen in the Figure 3a below. Here, each point only retains individual information without conserving the global or local interactions.

When the perplexity was raised to 25, which was the default value for tSNE computation on Seurat, it can conserve the local structure of the plot. This can be seen in the Figure 3b .

As the value of perplexity was increased, for example in Figure 3c and Figure 3d, for the value of 100 and 1000 respectively, the points in the plot began to become closer to each other, due to the increase in the number of nearest neighbours.

UMAP Optimisation

For the UMAP optimisation, the number of neighbours was adjusted to see how it would affect the plot produced for the dimensionality reduction. The number of neighbours, similar to the perplexity of tSNE, will capture the local details of the structure at a lower value but losing the global view of the overall structure. At a bigger value, the local details will be lost but it can capture global structure of the manifold [11].

At the lowest possible value of number of neighbours of two, a similar pattern like the lowest perplexity value on tSNE plot was produced. Only a small number of points were connected to each other and the plot taken a circular-like shape. This can be seen on Figure 4a below.

When the number of neighbours was increased five, the plot already took a shape that resembles the UMAP plot with default value of 30. Figure 4b shows the UMAP plot with number of neighbours of five.

Subsequent increment of the number of neighbours did not change the pattern or the shape of the plots drastically, except for making the points clustered in a more condensed manner. Figure 4c and Figure 4d show the UMAP plot with number of neighbours of 100 and 1000 respectively.

Clustering

For clustering using Seurat, the resolution of the cluster was adjusted in the optimisation process. The resolution parameter responsible to the size and the structure for each cluster [12]. At the default value of 0.8 for the resolution, 23 clusters were found in the dataset from UMAP plot. Which can be seen in Figure 5.

At a smaller value of resolution, the size of each cluster became bigger and fewer clusters were produced. For example, in Figure 6a, the resolution was set at 0.1, and only six clusters can be formed. As the resolution being increased, for example to 0.4 and 0.6, the number of clusters increased to 12 and 17 respectively, as shown in Figure 6b and Figure 6c.

At the resolution value of two, the number of clusters produced increased to 40 These can be seen in the Figure 6d above. Based on this, as the value of resolution increases, the number of clusters produced will increase.

Cell Typing

The cell typing process was based on the gene markers that were reported in the original paper from which the dataset was obtained [2]. Six cells were detected based on the gene markers, which are primitive hematopoietic stem and progenitor cells (HSPC), erythroid, basophil, megakaryocyte, megakaryocyte–erythroid progenitor (MEP), and myeloid. The clustering of these gene markers can be seen in the plots on Figure 7.

Scanorama

The PCA plots for the merged datasets on the Scanorama package indicated that there are some overlaps between the infected and uninfected samples, especially for the plots of principal components one and two, and principal component three and four. This can be seen on the plots of Figure 8. The shape of the plots looks identical for the two treatments but one is slightly shifted for both of these two plots. For the plots of principal component five and six, and principal component seven and eight, the two treatments seem to be overlapping with each other.

As for the tSNE and UMAP plots, clear separation between the infected and uninfected can be seen in Figure 9 below.

When the gene markers for cell typing were plotted, each gene was found to resides in two cluster regions, each for the respective treatment. This can be seen on Figure 10 below. Based on this, it is clear that there was separation between the two treatments that suggested the presence of batch effects or due to infection.

Once the integration and batch correction were applied to the dataset, tSNE and UMAP dimensionality reduction were plotted. Based on the plots on Figure 11 below, there was overlaps between the infected and uninfected samples on both tSNE and UMAP, while some of the regions only consist of one of the treatments. This occurrence could suggest that the integration and batch correction retained the biological differences.

Gene markers were then plotted and showed that for each gene, only specific clusters contain the specific genes for the respective cells. This can be seen on Figure 12 below. In addition to this, the gene marker for myeloid appeared to be lost during the integration process.

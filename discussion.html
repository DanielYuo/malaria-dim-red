Discussion

Batch Effect

Batch effect is the non-biological variations that can occurs in experimental data due to the differences in capturing times, handling personnel, and equipment [13]. This batch effect can confound the biological variations in the data and thus causing an inaccurate representation. Thus, it is crucial to perform a batch correction if there are any non-biological factors in the experiment.

Data Integration and Batch Correction

Data integration and batch correction aim to reduce the non-biological systemic variations in the data, while retaining as much of the biological variations as possible. For this process, there needs to be a balance between correcting the batch effect and retaining the biological information. Overcorrection would lead to some biological information to be lost. In this study, two integration and batch correction packages had been used to compare the effect on the data.

Seurat

The integration and batch correction from Seurat package uses CCA with anchors for the algorithm. The CCA was used to identify the conserved shared gene structure between different datasets that were integrated which in turn find a relationship between a single cell of a gene set from different datasets [14]. This helps in retaining the biological information across the different datasets.

The integration anchor is cell pairs that resulted from the mutual nearest neighbours (MNN) after the CCA had been applied. These cell pairs allow for the cellular relationship across the datasets to be retained for the integration process [7].

Scanorama

Scronorama took the approach akin to the panorama stitching computer vision algorithm by comparing the transcriptional profile of the cells across datasets and merged the overlap cells without affecting the cells that do not fit the profile [8]. One of the differences between Scanorama integration and Seurat, is that in Scanorama, the number of datasets with a matching cell can be as low as two when finding the MNN. Whereas in Seurat, all of the datasets must have the matching cell for the MNN computation. This can reduce the overcorrection of the integration.

Parameter Optimisation

Dimensionality Reduction

The two parameters that were focused in this study were the perplexity for tSNE and the number of neighbours for UMAP. These two parameters responsible in the number of the nearest for each point on the plot. Thus, at a smaller number, only a small number of points are considered as neighbours for a particular point, causing the points to cluster into small structure. The small value of perplexity and number of neighbours thus preserve the local structure of the dimensionality reduction as the clusters are more fine grained.

On the other hand, larger values of perplexity and number of neighbours recognised more points as neighbours for a particular point, causing more points to cluster together and being closer to each other. The structure thus reveals a more global view of the plot.

Clustering

The resolution parameter that was varied for the clustering involved in the determining the size and structure of the cluster. As the value of resolution increases, the number of clusters computed will increase at the expense of the size of the cluster, and the opposite is true [12].

When determining the value of the resolution, it is important to consider the number of the cells in the dataset, and the gene markers that can be found [15].
